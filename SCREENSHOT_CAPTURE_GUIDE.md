# Screenshot Capture Guide for SortBliss

Complete guide for capturing professional App Store and Google Play screenshots.

---

## Overview

**Objective:** Capture 6-8 high-quality screenshots showcasing SortBliss's core features and unique selling points.

**Target Devices:**
- iOS: iPhone 6.7" (iPhone 15 Pro Max) - 1290 x 2796 pixels
- Android: 1080 x 1920 pixels minimum

**Tools Needed:**
- Physical device OR iOS Simulator / Android Emulator
- Screenshot tool (built-in or third-party)
- Image editing software (Figma, Photoshop, Sketch)
- Optional: Frame.io, Appure, or similar screenshot framing tool

---

## Screenshot Strategy

### The 6-Screenshot Formula

1. **Hero Shot** - Main gameplay showing drag & drop
2. **Progression** - Multiple levels showing variety
3. **Daily Rewards** - 7-day calendar with streak
4. **Achievements** - Progress tracking and unlockables
5. **Customization** - Settings and theme options
6. **Social Proof** - Reviews or feature callout

**Optional 7th & 8th:**
7. Storefront showing IAP options
8. Premium features (3D containers, audio themes)

## Screenshot 1: Hero Shot - Core Gameplay

**Purpose:** Show main gameplay loop immediately

**Scene Setup:**
1. Launch app and start Level 3 or 4 (mid-difficulty)
2. Show 2-3 containers with items
3. Capture during drag action (item being dragged)
4. Ensure UI is clean (no debug overlays)

**What to Show:**
- Central pile with multiple items
- At least 2 containers (different colors)
- One item being dragged (mid-air)
- Score counter visible
- Level indicator showing "Level 3" or "Level 4"

**Composition:**
- Item being dragged should be in upper-middle of screen
- Containers at bottom third
- Central pile in middle
- Clean, uncluttered background

**Caption Overlay:**
"Intuitive Drag & Drop Gameplay"
or
"Relaxing Puzzle Sorting"

**Call-to-Action:**
- Arrow pointing to dragged item
- Subtle glow/highlight effect

## Caption Overlay Design

**Typography:**
- **Font:** System font (San Francisco for iOS, Roboto for Android) OR custom brand font
- **Size:** 48-72px for main text, 32-48px for subtext
- **Color:** White with dark shadow OR dark with light shadow (high contrast)
- **Alignment:** Center or left-aligned
- **Position:** Top third or bottom third of image

**Style Options:**
1. **Minimal:** Text only, no background
2. **Banner:** Text on semi-transparent bar
3. **Badge:** Text in rounded rectangle
4. **Outlined:** Bold stroke around text

**Best Practices:**
- Keep text to 5-7 words max
- Use action verbs (Drag, Sort, Unlock, Customize)
- Include benefit (not just feature)
- Maintain brand voice (relaxing, zen, satisfying)

## Capture Workflow

### Method 1: Using iOS Simulator

**Steps:**
1. Open Xcode
2. Launch iOS Simulator (iPhone 15 Pro Max)
3. Run SortBliss in simulator
4. Navigate to desired screen
5. Press `Cmd + S` to save screenshot
6. Find screenshot on Desktop
7. Verify resolution is 1290 x 2796

**Pros:**
- Easy to capture
- Perfect resolution
- Quick iteration

**Cons:**
- May not show device-specific features
- Performance may differ from real device

### Method 2: Using Physical Device

**iOS:**
1. Connect iPhone to Mac
2. Open QuickTime Player
3. File → New Movie Recording
4. Select iPhone as camera source
5. Perform actions in app
6. Take screenshots from QuickTime recording
7. Resize to 1290 x 2796 if needed

**Android:**
1. Enable USB Debugging
2. Connect device to computer
3. Use `adb` command: `adb shell screencap -p /sdcard/screenshot.png`
4. Or use Android Studio's screenshot tool
5. Verify resolution is 1080 x 1920 minimum

**Pros:**
- Realistic device appearance
- Real performance
- Authentic touch interactions

**Cons:**
- Requires physical device
- More setup time
- May need editing for perfect framing

### Method 3: Screen Recording + Frame Export

**Steps:**
1. Record screen while using app (iOS/Android built-in)
2. Export video to computer
3. Use video editing software to extract perfect frames
4. Export frames as high-res images
5. Edit and enhance in Photoshop/Figma

**Pros:**
- Capture dynamic moments
- Multiple screenshot options from one recording
- Natural user interactions

**Cons:**
- Extra editing required
- Video compression may reduce quality
- Time-consuming

## Post-Processing Workflow

**Tools:**
- **Photoshop** for advanced editing
- **Figma** for mockups and device frames
- **Canva** for quick caption overlays
- **Sketch** for Mac users (alternative to Figma)

**Standard Editing Process:**

1. **Import Screenshot**
   - Verify correct resolution
   - Check for any artifacts

2. **Clean Up**
   - Remove any unwanted UI elements
   - Adjust brightness/contrast
   - Enhance colors slightly (don't overdo)

3. **Add Device Frame** (optional)
   - Choose appropriate device
   - Center screenshot in frame
   - Add shadow/background

4. **Add Caption Overlay**
   - Write compelling text
   - Choose readable font
   - Ensure high contrast
   - Test at thumbnail size

5. **Add Visual Enhancements**
   - Arrows pointing to features
   - Glow effects on buttons
   - Subtle gradients
   - Icons or badges

6. **Export**
   - PNG format for best quality
   - Optimize file size (under 10MB)
   - Name consistently (screenshot_1.png, screenshot_2.png)

## A/B Testing Screenshots

**Recommended Approach:**
1. Create 2 variations of each screenshot
2. Test with small audience (TestFlight, Google Play Internal Testing)
3. Measure conversion rate (impressions → installs)
4. Choose winning variation
5. Roll out to full audience

**Variables to Test:**
- Caption wording ("Relaxing Puzzle" vs "Zen Sorting Game")
- Color scheme (light vs dark backgrounds)
- Device frames (with vs without)
- Screenshot order
- Feature highlights

**Tools:**
- SplitMetrics
- Storemaven
- App Radar

2. **Consistency** - Unified visual style across all shots

---

## Common Mistakes to Avoid

❌ **Too much text** - Keep captions short and impactful
❌ **Low resolution** - Always use maximum device resolution
❌ **Inconsistent style** - Maintain visual coherence
❌ **Static screenshots** - Show app in action
❌ **Cluttered composition** - Negative space is good
❌ **Poor contrast** - Ensure text is always readable
❌ **Outdated UI** - Re-capture after any UI changes
❌ **Wrong aspect ratio** - Verify dimensions before editing
❌ **Over-editing** - Keep it realistic and authentic
❌ **No clear hierarchy** - Lead viewer's eye through screenshot
