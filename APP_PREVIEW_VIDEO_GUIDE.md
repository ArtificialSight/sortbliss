# App Preview Video Production Guide for SortBliss

Complete guide for creating professional app preview videos for App Store and Google Play.

---

## Overview

**Objective:** Create a compelling 15-30 second app preview video showcasing SortBliss's core gameplay and unique features.

**Target Platforms:**
- **iOS:** App Preview (App Store)
- **Android:** Promo Video (Google Play)

**Video Specifications:**
- **Length:** 15-30 seconds (recommended: 20-25s)
- **Resolution:** 1080p minimum (1920 x 1080 or 1080 x 1920 for vertical)
- **Format:** MP4, MOV, or M4V
- **Frame Rate:** 30 fps minimum
- **Orientation:** Portrait (recommended) or Landscape
- **File Size:** Under 500MB

---

## Video Strategy

### The Perfect 25-Second Formula

**0-5s:** Hook - Show unique selling proposition
**5-15s:** Core Gameplay - Demonstrate main mechanics
**15-20s:** Features - Highlight key differentiators
**20-25s:** Call-to-Action - Encourage download

## Production Workflow

### Pre-Production (2-3 hours)

1. **Storyboard Creation**
   - Sketch each scene
   - Mark timing for each shot
   - Note camera movements
   - Plan text overlay positions

2. **Asset Preparation**
   - Ensure app is in demo-ready state
   - Prepare clean save file with ideal progress
   - Test all scenes for visual bugs
   - Export logo and branding assets

3. **Music Selection**
   - Choose background music (royalty-free)
   - Recommended sources:
     - Epidemic Sound
     - Artlist
     - Audio Jungle
     - YouTube Audio Library
   - Match music to brand (relaxing, uplifting, zen)

### Production (3-4 hours)

**Step 1: Screen Recording**

**iOS Method:**
1. Use built-in screen recording (Settings → Control Center → Screen Recording)
2. Or use QuickTime Player with connected iPhone
3. Or use iOS Simulator with screen capture

**Android Method:**
1. Use ADB screenrecord: `adb shell screenrecord /sdcard/demo.mp4`
2. Or use built-in screen recording (Android 10+)
3. Or use Android Studio emulator with record feature

**Recording Tips:**
- Record at 1080p minimum resolution
- Use 60fps for smoother motion
- Record each scene separately
- Do multiple takes of each scene
- Keep recordings longer than needed (trim in editing)

**Step 2: Capture Multiple Takes**
- Tutorial scene: 3-5 takes
- Gameplay progression: 2-3 takes per level
- Daily rewards: 2-3 takes
- Settings: 2 takes
- Opening/closing scenes: 2-3 takes

**Step 3: Export Best Takes**
- Review all footage
- Select best take of each scene
- Export to computer
- Organize in folders by scene

### Post-Production (4-6 hours)

**Video Editing Software Options:**
- **iMovie** (Mac, free, beginner-friendly)
- **Final Cut Pro** (Mac, professional)
- **Adobe Premiere Pro** (Mac/PC, professional)
- **DaVinci Resolve** (Mac/PC/Linux, free/professional)
- **CapCut** (Free, mobile-first, easy)

**Editing Process:**

1. **Import & Organize**
   - Import all video clips
   - Import music track
   - Import sound effects
   - Create bins/folders for organization

2. **Assembly Edit**
   - Place clips in timeline according to script
   - Rough timing (don't worry about precision yet)
   - Trim obvious excess

3. **Fine Edit**
   - Precise timing to match script
   - Add transitions (cross-dissolve recommended, 0.5s duration)
   - Speed up/slow down clips as needed
   - Ensure smooth flow between scenes

4. **Audio Mixing**
   - Add background music
   - Adjust music volume (-18dB to -12dB)
   - Add sound effects
   - Duck music under narration (if using voice-over)
   - Fade in music at start, fade out at end

5. **Text Overlays**
   - Add on-screen text per script
   - Font: San Francisco (iOS) or Roboto (Android)
   - Size: Large enough to read on mobile (48pt+)
   - Color: White with subtle shadow OR contrasting color
   - Animation: Simple fade in/out or slide up
   - Duration: 2-3 seconds per text element

6. **Visual Effects**
   - Add subtle zoom to maintain visual interest
   - Highlight important elements (glow, arrows)
   - Particle effects for celebration moments
   - Color correction for consistency

7. **Timing Refinement**
   - Ensure video is exactly 25 seconds (or target length)
   - Verify pacing feels natural
   - Check that text appears long enough to read
   - Ensure smooth transitions

8. **Quality Check**
   - Watch full video 5+ times
   - Check on mobile device
   - Verify audio levels
   - Look for visual glitches
   - Test with sound off (many viewers watch muted)

9. **Export**
   - Resolution: 1920 x 1080 (landscape) or 1080 x 1920 (portrait)
   - Format: MP4 (H.264 codec)
   - Frame rate: 30fps
   - Bitrate: 10-20 Mbps
   - File size: Under 500MB

## Text Overlay Design

**Typography:**
- **Font:** SF Pro Display (iOS) or Roboto (Android)
- **Size:** 60-80pt for main text, 40-50pt for subtext
- **Weight:** Semi-Bold or Bold
- **Color:** White (#FFFFFF) with dark shadow
- **Shadow:** 4px offset, 50% opacity black
- **Alignment:** Center (for most text)

**Animation:**
- Fade in: 0.3s
- Hold: 2-3s
- Fade out: 0.3s
- OR Slide up from bottom: 0.5s

**Positioning:**
- Main text: Top third or bottom third
- Avoid center (where gameplay happens)
- Leave breathing room around text

## Platform-Specific Requirements

### iOS App Preview

**App Store Connect Requirements:**
- **Devices:** iPhone, iPad (separate videos)
- **Orientations:** Portrait, Landscape
- **Length:** 15-30 seconds per preview
- **Maximum:** 3 previews per device size
- **Autoplay:** First 15 seconds play without sound

**Best Practices:**
- Front-load most important content (first 5 seconds)
- Design for silent viewing (clear text overlays)
- Use captions for narration
- Avoid rapid cuts (can be jarring)

**Upload Process:**
1. Log in to App Store Connect
2. Navigate to app version
3. Select device size (e.g., 6.5" iPhone)
4. Upload app preview video
5. Set poster frame (thumbnail)
6. Preview before submitting

### Android Promo Video

**Google Play Console Requirements:**
- **Length:** 30 seconds to 2 minutes
- **Format:** YouTube video
- **URL:** Public or unlisted YouTube link
- **Recommended:** 30-60 seconds (shorter is better)

**Best Practices:**
- Can be longer than iOS (up to 2 min)
- Include gameplay footage (required)
- Add channel branding (subscribe button)
- Use YouTube cards for website link

**Upload Process:**
1. Upload video to YouTube
2. Set to Public or Unlisted
3. Copy video URL
4. Paste in Google Play Console → Store Listing
5. Preview before publishing

## Advanced Techniques

### Motion Graphics

**Add Polish With:**
- **Lower thirds** - Name display for features
- **Callout boxes** - Highlight specific UI elements
- **Arrow animations** - Direct viewer attention
- **Icon animations** - Feature icons flying in

**Tools:**
- **After Effects** - Professional motion graphics
- **Motion** (Mac) - Simpler alternative
- **Keynote** - Surprising option for basic animations

### Screen Mockups

**Instead of raw screen recording:**
- Place gameplay in device mockup
- Add background environment
- Show multiple devices
- Create cinematic look

**Tools:**
- **Rotato** - 3D device mockups ($59)
- **Previewed** - Animated mockups (free/$)
- **Mockuuups Studio** - Quick mockups

### Transitions

**Effective Transition Types:**
- **Cross-dissolve** (most versatile)
- **Wipe** (directional movement)
- **Push** (one scene pushes out another)
- **Zoom** (scale transition)

**Avoid:**
- **Star wipes, page curls** (too cheesy)
- **Random transitions** (inconsistent)
- **Long transitions** (>0.5s wastes time)

## Common Mistakes to Avoid

❌ **Too long** - Over 30 seconds loses attention
❌ **Too much text** - Viewers can't read it all
❌ **Rapid cuts** - Causes confusion, eye strain
❌ **Wrong aspect ratio** - Black bars look unprofessional
❌ **No audio** - Silent videos feel incomplete
❌ **Loud music** - Overwhelming, startling
❌ **Poor quality** - Blurry, pixelated footage
❌ **Misleading** - Shows features not in app
❌ **No CTA** - Viewers don't know what to do next
❌ **Off-brand** - Doesn't match app's tone/style
